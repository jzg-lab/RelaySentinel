后端模块

建议做成单体服务里的清晰模块，先别拆微服务。

01 Scheduler 按 10 分钟、1.5 小时和上游自定义间隔触发检查。
02 Adapters 上游适配 New API/Sub2API；号池高级额度来源再接 CLIProxyAPI/CPA。
03 Quota Engine 归一化余额、额度、5H/7D 快照和预测。
04 Alert Engine 阈值判断、冷却、恢复、消息模板。
05 Notifier 企业微信、飞书、Telegram、自定义 webhook。

接口草案

REST + Server-Sent Events 足够。V1 不需要 GraphQL。

GET /api/dashboard/summary

工作台总览：低余额数、账号健康、5H/7D 可用时长。

POST /api/upstreams

手动添加外部上游，仅 New API/Sub2API。

POST /api/pools

手动添加自己的中转站或号池，默认 New API/Sub2API。

POST /api/pools/:id/quota-sources

给号池接入 CLIProxyAPI/CPA 等高级额度来源。

GET /api/pools/:id/snapshots

查询账号成功/失败、5H/7D 快照和可用时长预测。

GET /api/events/stream

SSE 推送检查完成、告警触发和恢复事件。
